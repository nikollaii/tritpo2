# 1 Введение
<a name="intro"/>

## 1.1 Название продукта
<a name="product_name"/>
CryptoWallet

## 1.2 Назначение документа
<a name="appointment"/>
В этом документе описаны функциональные и нефункциональные требования к веб-сервису «CryptoWallet» для мобильных телефонов.
Этот документ предназначен для команды, которая будет реализовывать и проверять корректность работы веб-сервиса.

## 1.3 Бизнес-требования
<a name="business_requirements"/>

### 1.3.1 Исходные данные
<a name="initial_data"/>
Криптокошелёк – набирающая популярность уже который год бизнес-модель. Ведь так или иначе, здесь идет игра на финансовых желаниях и мечтах человека.
В первую очередь (для пользователей) криптокошелёк – это место, где предоставляется возможность обмениваться электронной валютой. Можно получать переводы электронных денежных стредств. А также можно отправлять криптовалюту другим пользователяв просто узнав их адрес кошелька определённого токена, который вы хотите отправить. Криптокошелек - это будущее, так как сама децентрализованная идея криптовалют даёт пользователям полную безопасность и анонимность.

### 1.3.2 Возможности бизнеса
<a name="business_opportunities"/>
Многие люди желают иметь криптокошелек, потому что на нем они могут безопасно хранить свои активы, а также выполнять безопасные переводы любых сумм. Данное приложение необходимо пользователям, чтобы полностью организовывать и контролировать свои взаимодействия с криптоваютой. Интерфейс, спроектированный с учётом всех особенностей похожих технологий, и дополнение веб-сервиса подробной инструкцией позволят увеличить количество людей, использующих данный сервис.
Отправление/Получение криптовалюты будет осуществлять по специальному алгоритму, который включает в себя полную безопасность и анонимность отправителя/получателя.

### 1.3.3 Границы проекта
<a name="project_boundary"/>
Веб-сервис "CryptoWallet" позволит зарегистрированным пользователям создать свой криптокошелёк, хранить свои криптовалюты, такие как: BTC, ETH, а также можно выполнять переводы и аналогично получать их на свой кошелёк. Так же можно оформить подписку, после чего будут доступны пониженные комиссии за переводы и отключение рекламы

## 1.4 Аналоги
<a name="analogues"/>
Trust Wallet — это быстрый и безопасный мульти-крипто-кошелек с поддержкой Binance DEX, разработанный для простоты использования и идеально подходящий для хранения криптоактивов. 

Matbea - это надежный кошелек и сервис обмена с тысячами пользователей. С Matbea вы можете легко управлять своими ежедневными транзакциями в биткоинах и альткоинах, а так же безопасно хранить их в одном месте.

### 1.4.1 Отличия от аналогов
В отличие от Trust Wallet и Matbea пользователи веб-сервиса CryptoWallet имеют пониженные комиссии на переводы и изменение темы интерфейса (светлая/темная).

## 2.3 Характеристики пользователей
<a name="user_specifications"/>

### 2.3.1 Классы пользователей
<a name="user_classes"/>
1)Зарегистрированные пользователи - пользователи, которые зарегестрировались в приложение и создали свой кошелёк. Имеют доступ к начальному функционалу
2)VIP пользователи - пользователи, которые оформили подписку. Имеют доступ к полному функционалу.

### 2.3.2 Аудитория приложения
<a name="application_audience"/>

#### 2.3.2.1 Целевая аудитория
<a name="target_audience"/>

Приложение расчитано на совершеннолетних пользователей возрастной категории(18-40) лет, обладающие умением взаимодействовать с телефоном и знаниями в области криптовалюты.

#### 2.3.2.1 Побочная аудитория
<a name="collateral_audience"/>

Люди старше 40 лет, обладающие вышеперечисленными качествами

### 2.4 Предположения и зависимости
<a name="assumptions_and_dependencies"/>
1. Веб-сервис должен иметь возможность обрабатывать значительное количество запросов в секунду;
2. Допустимое время ответа сервера не должно превышать 2 секунды;
3. На сервисе ожидается большое количество пользователей;
4. Требуется хранить значительное количество информации об кошельках пользователей и их балансах.

# 3 Cистемные требования
<a name="system_requirements"/>

## 3.1 Функциональные требования
<a name="functional_requirements"/>

### 3.1.1 Основные функции
<a name="main_functions"/>

#### 3.1.1.1 Вход пользователя в приложение
<a name="user_logon_to_the_application"/>
Описание. Пользователь не имеет возможность использовать приложение без создания собственного кошелька либо войдя в свою учётную запись.

| Функция | Требования | 
|:---|:---|
| Вход в приложение без создания собственного кошелька | Приложение не предоставляет пользователю возможность войти в приложение без исполььзования кошелька |
| Регистрация нового пользователя | Приложение должно запросить у пользователя ввести свою почту или номер телефона для создания учётной записи.|
| Пользователь с такой почтой(телефоном) уже существует | Приложение должно известить пользователя об ошибке регистрации и запросить ввод уже другой почты или нормера телефона. Пользователь должен либо ввести почту/телефон, либо отменить действие |
| Вход зарегистрированного пользователя в приложение | Приложение должно предоставить пользователю список почт (телефонов) уже зарегестрированных пользователей. Пользователь должен либо выбрать из списка своё имя (псевдоним), либо отменить действие |

#### 3.1.1.2 Настройка профиля зарегистрированного пользователя
<a name="setting_up_the_profile_of_the_active_user"/>
Описание. Зарегистрированный пользователь имеет возможность единоразово сгенирировать адрес своего кошелька для каждого токена.

| Функция | Требования | 
|:---|:---|
| Создание адреса кошельков пользователя | Приложение должно предоставить зарегистрированному пользователю возможность один раз создать адрес кошельков для BTC и ETH|

#### 3.1.1.3 Опрос пользователя о предпочтениях используемой криптовалюты
<a name="user_asking"/>
Описание. Зарегистрированный пользователь имеет возможность указать приложению свои предпочтения, которые в дальнейшем обработаются приложением.

| Функция | Требования | 
|:---|:---|
| Заполнение предпочтений | Приложение должно предоставить зарегистрированному пользователю поле для ввода информации о предпочтениях. Пользователь должен либо ввести информацию и подвердить действие, либо пропустить его |

#### 3.1.1.4 Просмотр информации об балансе
<a name="view_information_about_an_individual_anket"/>
Описание. Зарегистрированный пользователь имеет возможность просматривать информацию о балансе

| Функция | Требования | 
|:---|:---|
| Просмотр информации о балансе | Приложение должно предоставить пользователю возможность увидеть информацию о балансе на главной странице |

#### 3.1.1.5 Выход зарегистрированного пользователя из учетной записи
<a name="active_user_change"/>
Описание. Зарегистрированный пользователь имеет возможность выйти из своего профиля.

Требование. Приложение должно предоставить зарегистрированному пользователю возможность выйти из учётной записи с возвратом к окну входа в приложение.

#### 3.1.1.6 Регистрация нового пользователя после входа в приложение
<a name="add_new_user"/>
Описание. Анонимный пользователь имеет возможность зарегистрироваться в приложении.

Требование. Приложение должно предоставить анонимному пользователю возможность [зарегистрироваться в приложении]

#### 3.1.1.7 Отправка перевода
<a name="watch_anket"/>
Описание. Зарегистрированный пользователь имеет возможность отправлять переводы другим пользователям по адресу их кошельков

Требование. Приложение должно предоставить анонимному пользователю возможность отправлять переводы другим пользователям по адресу их кошельков, отобразив интерфейс переводов

#### 3.1.1.8 Просмотр истории транзакций
<a name="download_anket"/>
Описание. Зарегистрированный пользователь имеет возможность просмотра истории транзакций

Требование. Приложение должно предоставить зарегистрированному пользователю возможность просмотра истории транзакций, отобразив графический интерфейс просмотра истории транзакций

### 3.1.2 Ограничения и исключения
<a name="restrictions_and_exclusions"/>
1. Приложение работает только при наличии подключения к Интернету;
2. Приложение функционирует корректно только при корректной работе сервиса CryptoWallet 

## 3.2 Нефункциональные требования
<a name="non-functional_requirements"/>

### 3.2.1 Атрибуты качества
<a name="quality_attributes"/>

#### 3.2.1.1 Требования к удобству использования
<a name="requirements_for_ease_of_use"/>
1. Доступ к основным функциям приложения не более чем за две операции;
2. Все функциональные элементы пользовательского интерфейса имеют названия, описывающие действие, которое произойдет при выборе элемента;
3. Пошаговая инструкция использования основных функций приложения отображена в справке;
4. Понятный графический интерфейс
5. Обновление информации о курсах валют происходит каждые 30 секунд в фоновом режиме

#### 3.2.1.2 Требования к безопасности
<a name="security_requirements"/>
Приложение предоставляет возможность просмотра и редактирования профиля только зарегистрированного пользователя.
Приложение гарантирует безопасность личных данных пользователей.

### 3.2.2 Внешние интерфейсы
<a name="external_interfaces"/>
Окна приложения удобны для использования пользователями с плохим зрением:
  * размер шрифта не менее 14пт;
  * функциональные элементы контрастны фону окна.

### 3.2.3 Ограничения
<a name="restrictions"/>
1. Приложение реализовано на платформе Android Studio;
2. Профиль пользователя хранится в файле с расширением XML, название файла совпадает с почтой (телефоном);
3. Информация о кошельках и балансе хранится в блокчейне.
4. Приложение не работает при отсутствии подключения к Интернету;
5. Скорость работы приложения зависит от скорости подключения пользователя.
